## 📜 로블록스 개발 3대 헌법 (핵심 지침)
*로블록스 공식 개발자 포럼의 검증된 가이드라인을 기반으로 수립되었습니다: [How to actually make a popular game](https://devforum.roblox.com/t/how-to-actually-make-a-popular-game/3115791).*

로블록스의 추천 알고리즘을 뚫고 상위 차트에 진입하기 위해, 본 프로젝트에서 빌드하는 모든 게임은 아래의 3대 지표를 무조건 충족해야 합니다.

### 1. 목표 몰입도 및 플레이 타임 (Playtime)
* **기준 수치**: 유저당 평균 플레이 타임 **9분 ~ 15분 이상** (Excellent 등급) 달성 목표.
* **전략**: 진입 후 최초 5분 이내 이탈을 막는 초반 온보딩 설계.

### 2. 리텐션 유도형 코어 루프 (Core Loop)
* **기준 수치**: 유저가 반복 행동에 질리지 않는 유연한 순환 경제 구조 구축.
* **전략**: 펫 거래 경제, 환생 시스템, 3분 주기 보상 등을 통한 지속적인 단기/장기 목표 부여.

### 3. 데이터 기반 피벗 및 냉정한 평가 (Data-Driven Pivot)
* **기준 수치**: 출시 후 **첫 10일 ~ 25일** 동안 축적된 데이터가 게임의 수명을 결정함.
* **전략**: 초기 정밀 타겟팅 광고를 집행하여 알고리즘 트래픽 지표를 검증하고, 기준 미달 시 감정을 배제하고 초고속 피벗(Pivot) 감행.

## 🔍 2. 에셋별 심층 역설계 분석

### 🌿 Grow a Garden 2 (`10200395747`)
* **기획적 역설계 (도파민 루프와 소셜 약탈)**:
  * **초반 도파민 스파이크**: 기본 정착 배지(`Carrot!`, 23.6%) 대비 프리미엄 보상 배지(`First Mutation!` 15.4%, `Golden!` 13.8%, `Rainbow!` 11.0%)의 획득률이 이례적으로 높습니다. 이는 유저가 완전히 안착하기 전에 고성능 시각적 보상을 퍼부어 초반 이탈률을 강제로 억제하는 온보딩 빌드입니다.
  * **소셜 마찰**: 전체 유저의 12.7%가 `Stole a Fruit!` 배지를 보유하고 있습니다. 방치형 타이쿤의 정적이고 지루한 한계를 경쟁과 복수라는 소셜 인터랙션으로 깨부수며 45만 동접의 강력한 방화벽을 형성합니다.
* **숨겨진 BM 구조 추론**:
  * 초고희귀 펫 배지인 `OMG its MEGA!`의 획득률이 **0.0%**를 기록 중입니다. 외부 오픈 API에는 잡히지 않는 **인게임 커스텀 UI 기반의 헤비 과금러(Whale) 타겟 뽑기(Gacha) 모델**이 게임의 진짜 핵심 매출을 견인하고 있습니다.

### ⚔️ [BLACK DEATH] Jujutsu Shenanigans (`3508322461`)
* **기획적 역설계 (극단적 하드코어와 고인물 락인)**:
  * **1.7%의 역설**: 59억 회가 넘는 메가 히트작임에도 `Your First Kill` 배지 획득률은 단 1.7%에 불과합니다. 진입 유저 100명 중 98명이 1킬도 못 따고 도망치는 구조로, 일반적인 PM 지표 기준으로는 실패한 기획입니다.
  * **매니아층의 하드캐리**: 하지만 이 장벽을 뚫고 잔존한 1.7%의 하드코어 유저들이 미친 수준의 체류 시간과 판수를 기록합니다. 이 팬덤이 플랫폼 추천 알고리즘의 플레이 타임 가중치를 하드 캐리하며 동접 18만을 견고하게 방어합니다.
* **숨겨진 BM 구조 추론**:
  * 판매 중인 웹 공식 게임패스가 **0개**인 구조입니다. 모든 결제 동선이 인게임 루아(Luau) 스크립트로 격리되어 있으며, 서버 주도형 즉시 소비 상품(전 서버 타겟 맵 변경권, 캐릭터 각성 이펙트, 커스텀 스킨 및 감정표현)으로 감성적 과금을 교묘하게 유도합니다.

## 🧠 3. Day 2 데이터 교차 검증 절차

1. **백엔드 원천 데이터 조회 (API 검증)**: 로블록스 공식 웹 엔드포인트 URL 주소에 해당 Universe ID를 대입하여 원천 JSON 데이터(`playing`, `visits`, `winRatePercentage`)를 실시간 상호 대조합니다.
2. **인게임 클라이언트 테스트 (인게임 BM 검증)**: PC/모바일 클라이언트로 실제 게임에 접속해 상점 UI를 확인합니다. 모든 결제 시스템이 `MarketplaceService:PromptProductPurchase()` 스크립트로 래핑되어 웹 크롤러를 우회하고 있음을 직접 눈으로 검증합니다.
3. **개발자 포럼 패러다임 분석 (생태계 트렌드 검증)**: 공식 개발자 포럼(`devforum.roblox.com`)의 최상위 크리에이터 스레드를 분석하여, 웹 게임패스 배제 및 인게임 소소성 개발자 상품(Developer Products) 전면 배치가 탑 어닝 진입의 주류 공식임을 방증 자료로 확보합니다.
